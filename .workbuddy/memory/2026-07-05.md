## AI 无法回答问题排查（00:40 - 00:55）

### 问题现象
前端对话报"下游服务不可用"（错误码 5002），AI 不输出任何内容。

### 根因 1：DashScope SDK 兼容性 bug（已修复）
`backend/ai_service/adapter/dashscope_adapter.py:96` 用 `getattr(getattr(response, "output", None), "embeddings", None)` 访问 embeddings，但 dashscope SDK 1.26.2 的 `response.output` 返回的是 **dict** 不是对象，导致 `getattr(dict, "embeddings")` 返回 None → 报"向量化服务返回空结果" → 文档入库失败 → 知识库为空 → AI 返回 5003。
**修复**：改为先判断 `isinstance(output, dict)` 再 `.get("embeddings")`，兼容 dict 和对象两种返回。修复后直接调 ai_service embedding 接口正常（0.5 秒返回，维度 1024）。

### 根因 2：httpx 连接池死连接（已修复，待重启验证）
`backend/common/http_client.py` 的 `httpx.AsyncClient` 默认开 keep-alive，连接池复用连接。当下游服务 reload 重启后（如 ai_service 因代码改动触发热重载），池里旧连接失效，gateway/knowledge_service/chat_service 复用死连接 → `httpx.RequestError`（err 为空）→ 报 5002 "下游服务不可用"。
**修复**：`http_client.py:create_client` 改为 `limits=httpx.Limits(max_keepalive_connections=0)` 禁用 keep-alive，每次请求新建连接。代价是轻微性能损失，但彻底避免死连接问题。

### 排查路径
1. 直接 curl ai_service:8004/api/ai/chat → 返回 5003 "知识库为空"（说明 ai_service 本身正常，问题是知识库没文档）
2. 尝试上传文档 → gateway 超时 30 秒返回 5002
3. 直接 curl ai_service:8004/api/ai/embedding → 0.5 秒正常返回（说明 ai_service 没问题）
4. gateway 日志：`转发请求异常: POST ...8003... err=` （err 为空 = httpx RequestError）
5. 定位：httpx 连接池缓存死连接
