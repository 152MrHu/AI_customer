# 软件需求说明书

## 1. 项目概述

### 1.1 项目名称

AI 智能客服系统

### 1.2 项目简介

本项目是一个基于 RAG（检索增强生成）技术的智能客服系统，采用微服务架构设计。系统通过知识库文档管理、语义向量检索、大语言模型生成等能力，为用户提供准确、可靠、可溯源的自动化问答服务。同时支持通用助手模式（联网搜索增强）和人工客服转接，实现 AI 与人工协同工作。

### 1.3 项目目标

- 构建一个支持多知识库管理的智能问答平台
- 实现基于 RAG 的精准问答，回答可溯源到具体文档
- 提供 SSE 流式输出，带来流畅的对话体验
- 支持多轮对话记忆，上下文理解能力强
- 提供人工客服转接通道，AI 与人工协同
- 确保系统安全性，包括 Prompt 注入防护、限流等

### 1.4 适用范围

本系统适用于以下场景：
- 企业制度/规章问答
- 产品手册/技术文档问答
- 校园办事流程咨询
- 常见问题（FAQ）自动回答
- 电商售前售后咨询

## 2. 用户角色分析

### 2.1 普通用户（User）

**职责**：使用系统进行智能问答

**核心需求**：
- 创建会话并发起 AI 对话
- 在知识库模式和通用助手模式之间切换
- 查看历史会话和消息
- 对 AI 回答进行赞/踩反馈
- 请求转接人工客服

### 2.2 客服人员（Agent）

**职责**：接管 AI 无法处理的复杂问题

**核心需求**：
- 查看待处理的转接工单
- 认领工单并进入会话
- 在线与用户实时交流
- 解决问题后关闭工单

### 2.3 管理员（Admin）

**职责**：系统管理与知识库维护

**核心需求**：
- 创建和管理知识库
- 上传、查看、删除文档
- 查看文档入库状态
- 管理用户（创建/编辑/删除/角色分配）
- 创建客服账号
- 重置用户密码

## 3. 功能需求

### 3.1 用户服务模块

#### FR-USER-001 用户注册

- **描述**：新用户通过用户名、手机号、密码完成注册
- **输入**：用户名（2-50 字符）、手机号（11 位）、密码（8-32 字符，含字母和数字）
- **输出**：注册成功提示，自动跳转登录页
- **约束**：用户名和手机号不可重复

#### FR-USER-002 用户登录

- **描述**：用户通过用户名/手机号 + 密码登录系统
- **输入**：账号（用户名或手机号）、密码、记住我选项
- **输出**：JWT Token（默认 24h，记住我 168h）
- **约束**：5 次失败后锁定 30 分钟

#### FR-USER-003 退出登录

- **描述**：用户安全退出，Token 加入黑名单
- **约束**：退出后 Token 立即失效

#### FR-USER-004 个人信息管理

- **描述**：查看和编辑个人资料
- **输入**：昵称、邮箱、手机号
- **约束**：手机号不可与他人重复

#### FR-USER-005 密码管理

- **描述**：修改个人密码（需验证旧密码）
- **输入**：旧密码、新密码
- **约束**：新密码须符合强度要求

#### FR-USER-006 自助重置密码

- **描述**：忘记密码时通过用户名和手机号验证后重置
- **输入**：用户名、手机号、新密码
- **约束**：3 次/小时 频率限制

#### FR-USER-007 用户管理（管理员）

- **描述**：管理员查看、创建、编辑、删除用户
- **功能**：用户列表（支持关键词/状态/角色筛选）、创建客服账号、修改角色、禁用/启用
- **约束**：管理员不能操作自己

### 3.2 对话服务模块

#### FR-CHAT-001 会话管理

- **描述**：用户创建、查看、删除会话
- **功能**：
  - 创建会话：指定知识库和对话模式（kb/assistant）
  - 会话列表：分页展示，支持关键词搜索
  - 会话详情：查看消息历史
  - 删除会话：软删除
- **约束**：每用户最多 100 个会话，超限自动删除最早会话

#### FR-CHAT-002 AI 消息发送（SSE 流式）

- **描述**：用户发送消息，系统通过 SSE 流式返回 AI 回答
- **输入**：消息文本（最多 2000 字符）
- **输出**：SSE 事件流（帧类型：token / sources / done / error）
- **约束**：10 次/分钟 频率限制

#### FR-CHAT-003 双模式对话

- **知识库模式（kb）**：
  - 基于指定知识库进行语义检索
  - 相似度 < 阈值（0.5）的结果被过滤
  - 知识库为空或无相关结果时明确告知用户
  - 回答附文档引用来源

- **通用助手模式（assistant）**：
  - 不依赖知识库，启用 DashScope 联网搜索
  - 可回答更广泛的问题
  - 未验证信息会标注

#### FR-CHAT-004 会话自动标题

- **描述**：会话创建后，系统将首条用户消息的前 20 字作为会话标题

#### FR-CHAT-005 多轮对话记忆

- **描述**：系统保留最近 20 条消息作为上下文传给 LLM
- **约束**：不同会话的上下文相互隔离

### 3.3 消息反馈模块

#### FR-FEEDBACK-001 消息反馈

- **描述**：用户对 AI 回答进行赞/踩评价
- **输入**：评分（1=赞，0=踩）、可选备注（最多 500 字符）
- **约束**：同一用户对同一消息只能提交一次，重复提交覆盖之前评价

### 3.4 人工转接模块

#### FR-HANDOFF-001 创建转接工单

- **描述**：用户在 AI 无法解决问题时请求转接人工
- **输入**：转接原因（最多 500 字符）
- **约束**：仅会话拥有者可创建

#### FR-HANDOFF-002 工单管理（客服/管理员）

- **描述**：客服查看和管理工单
- **功能**：
  - 工单列表（支持状态筛选、claimed_by=me 查看自己工单）
  - 待处理工单数量（通知角标）
  - 认领工单
  - 解决工单（填写解决方案）

#### FR-HANDOFF-003 工单关闭（用户）

- **描述**：用户关闭自己的已解决工单
- **约束**：仅 resolved 状态可关闭，仅工单所有者可操作

#### FR-HANDOFF-004 客服发送消息

- **描述**：已认领工单的客服可通过 SSE 流式发送消息到对应会话

### 3.5 知识库管理模块

#### FR-KB-001 知识库 CRUD

- **描述**：管理员创建、查看、删除知识库
- **功能**：
  - 创建知识库：名称（必填，唯一）、描述（可选）
  - 知识库列表：分页展示
  - 删除知识库：同步删除所有关联文档和向量数据

#### FR-KB-002 文档管理

- **描述**：管理员上传、查看、删除文档
- **功能**：
  - 上传文档：通过 multipart/form-data
  - 文档列表：分页，支持文件名关键词和状态筛选
  - 删除文档：同步删除向量数据

- **支持格式**：PDF、DOCX、TXT、MD、CSV
- **大小限制**：20MB
- **安全校验**：Magic bytes 验证、文件名清洗、同名文件检查

#### FR-KB-003 文档自动入库

- **描述**：文档上传后系统自动完成入库流程
- **流程**：解析文本 → 切块（size=600, overlap=100） → 调用 AI 服务向量化 → 写入向量存储 → 更新文档状态
- **状态**：pending（待处理） → ready（入库完成） / failed（入库失败）
- **超时**：120 秒整体超时保护

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| LLM 首 Token 响应 | ≤ 5 秒 |
| AI 服务整体超时 | 30 秒 |
| 文档入库超时 | 120 秒 |
| 并发会话数 | 支持 50+ 用户同时对话 |
| 向量检索响应时间 | ≤ 1 秒 |
| 页面首次加载 | ≤ 3 秒 |

### 4.2 安全需求

| 类别 | 要求 |
|------|------|
| 认证 | JWT + Token 黑名单机制 |
| 密码 | bcrypt 哈希（rounds=12），至少 8 位含字母数字 |
| 传输 | 密码通过 HTTPS 传输（生产环境） |
| 注入防护 | Prompt 注入模式检测、控制字符过滤 |
| 文件安全 | Magic bytes 校验、文件名清洗、类型白名单、大小限制 |
| 限流 | AI 问答 10次/分钟、登录注册 20次/分钟/IP |
| 登录锁定 | 5 次失败锁定 30 分钟 |
| 访问控制 | 管理接口仅管理员可访问，客服接口仅客服/管理员可访问 |
| 服务隔离 | 内部服务绑定 127.0.0.1，仅网关对外暴露 |
| 错误处理 | 不向客户端泄露内部错误详情 |

### 4.3 可用性需求

| 指标 | 要求 |
|------|------|
| AI 服务降级 | AI 不可用时给出友好提示，不影响其他功能 |
| 服务健康检查 | 所有服务提供 /health 端点 |
| 启动恢复 | knowledge_service 启动时自动重试 pending/failed 文档 |

### 4.4 可维护性需求

| 指标 | 要求 |
|------|------|
| 日志 | 统一日志格式，支持控制台 + 文件输出，按天滚动 |
| 请求追踪 | X-Request-Id 透传，便于跨服务追踪 |
| 配置管理 | 统一 .env 配置，pydantic-settings 读取 |
| 模型可替换 | LLM 适配器模式，切换模型不影响业务服务 |

- 操作系统：Windows 10+、Linux、macOS

## 5. 核心业务流程

### 5.1 AI 问答流程（知识库模式）

```
用户发送消息
    ↓
Gateway 鉴权 + 限流
    ↓
Chat Service 接收请求
    ↓
保存用户消息到 MySQL
    ↓
调用 AI Service /api/ai/chat
    ↓
AI Service: 输入安全清洗（Prompt注入检测）
    ↓
AI Service: 检查知识库是否为空
    ↓
AI Service: query 向量化（调用 DashScope）
    ↓
AI Service: 远程向量检索（调用 Knowledge Service）
    ↓
AI Service: 相似度过滤（threshold=0.5）
    ↓
AI Service: 构建 Prompt（知识库内容 + 对话历史 + 用户问题）
    ↓
AI Service: 调用 LLM 流式生成（DashScope）
    ↓
逐 token → SSE 帧流式返回到 Chat Service → Gateway → 浏览器
    ↓
流结束：AI Service 返回 sources + done 帧
    ↓
Chat Service 保存完整 AI 回复到 MySQL
```

### 5.2 AI 问答流程（通用助手模式）

```
用户切换到 assistant 模式发送消息
    ↓
Gateway 鉴权 + 限流
    ↓
Chat Service → AI Service
    ↓
AI Service: 使用 messages 格式调用 DashScope
    ↓
开启 enable_search=True（联网搜索增强）
    ↓
逐 token → SSE 流式返回
    ↓
无引用来源（sources=[]）
```

### 5.3 文档入库流程

```
管理员上传文档
    ↓
文件安全校验（类型、大小、Magic bytes、文件名）
    ↓
保存文件到 uploads/ 目录
    ↓
创建文档记录（status=pending）
    ↓
asyncio.create_task() 异步后台任务
    ↓
解析文本（按文件类型选择解析器）
    ↓
切块（size=600, overlap=100）
    ↓
调用 AI Service /api/ai/embedding 向量化
    ↓
写入 SQLite 向量存储（vec_chunks 表）
    ↓
更新文档状态：status=ready, chunk_count, processed_at
    ↓
更新知识库 document_count + 1
```

### 5.4 人工转接流程

```
用户请求转人工
    ↓
创建工单（status=pending）
    ↓
客服查看待处理工单列表
    ↓
客服认领工单（status=pending → claimed）
    ↓
客服进入会话，发送消息
    ↓
问题解决，客服提交解决方案（status=claimed → resolved）
    ↓
用户确认关闭（status=resolved → closed）
```

## 6. AI 功能需求说明

### 6.1 RAG 检索增强生成

- **检索方式**：基于 query 的语义向量检索
- **向量模型**：阿里云 DashScope text-embedding-v3
- **相似度计算**：余弦相似度
- **检索数量**：Top-5（可配置）
- **相似度阈值**：0.5（低于此值的结果被过滤）
- **LLM 模型**：Qwen-Turbo（可配置）

### 6.2 联网搜索增强

- **搜索模型**：Qwen-Plus（支持 enable_search）
- **搜索策略**：max（最大搜索范围）
- **来源标注**：enable_source=True，返回搜索结果来源

### 6.3 Prompt 管理

- **知识库模式 Prompt**：严格限定基于知识库内容回答，不编造信息
- **通用助手模式 Prompt**：友好助手，结合搜索结果回答
- **注入防护**：14 种注入模式检测，控制字符清理

### 6.4 模型适配器

- **架构**：LLM 适配器抽象基类 + 工厂模式
- **当前实现**：DashScopeAdapter（通义千问）
- **扩展预留**：OpenAI 适配器（工厂中已预留接口）
- **解耦设计**：业务服务不直接调用模型 API，统一通过 ai_service
