# 项目进度计划表

## 1. 项目基本信息

| 项目 | 内容 |
|------|------|
| 项目名称 | AI 智能客服系统 |
| 技术栈 | Python/FastAPI + React + MySQL + Redis + DashScope |
| 架构模式 | 微服务架构（5 服务 + 1 网关） |
| 开发周期 | 2 周 |
| 团队规模 | 4 人 |

## 2. 小组成员与职责分工

| 姓名 | 角色 | 职责范围 | 负责服务/模块 |
|------|------|----------|-------------|
| 组长A | 组长 / 架构与公共支撑 | 架构设计、公共模块、网关、项目协调 | gateway, common, 文档统筹 |
| 成员B | 身份与用户域负责人 | 用户注册登录、权限管理、客服管理 | user_service, 前端登录/注册 |
| 成员C | 核心业务域负责人 | 会话管理、消息反馈、人工转接 | chat_service, 前端聊天/管理 |
| 成员D | AI 与知识库负责人 | 知识库管理、文档入库、RAG问答、向量检索 | knowledge_service, ai_service, 前端知识库管理 |

### 2.1 分工职责明细

#### 组长A — 架构与公共支撑
- **负责组件**：`common/`, `gateway/`, `sql/`, `scripts/`
- **负责前端**：MainLayout, App.jsx, request.js
- **文档**：概要设计、详细设计、课程设计报告
- **重点任务**：
  - 搭建公共模块（config, database, redis, jwt, security, response, exceptions）
  - 实现网关（中间件栈：CORS → 日志 → JWT → 限流 → 代理转发）
  - 定义统一响应格式和错误码
  - 接口协议与公共依赖约定

#### 成员B — 身份与用户域
- **负责服务**：`user_service/`
- **负责前端**：Login.jsx, Register.jsx, Profile.jsx, AdminLayout.jsx, UserManage.jsx
- **文档**：数据库设计说明书（用户相关表）
- **重点任务**：
  - 用户注册/登录/登出
  - JWT Token 签发与验证集成
  - 密码安全（bcrypt、强度校验、登录锁定）
  - 用户管理（列表、角色、状态、删除）
  - 客服账号管理

#### 成员C — 核心业务域
- **负责服务**：`chat_service/`
- **负责前端**：Chat.jsx, SessionList.jsx, MessageList.jsx, MessageInput.jsx, SourceViewer.jsx, AgentWorkspace.jsx, AgentChat.jsx
- **文档**：测试报告
- **重点任务**：
  - 会话 CRUD（创建/列表/详情/删除/100条限制）
  - 消息发送与 SSE 流式接收
  - 消息持久化存储
  - 消息反馈（赞/踩）
  - 人工转接工单（创建/认领/解决/关闭）
  - 客服消息发送

#### 成员D — AI 与知识库
- **负责服务**：`knowledge_service/`, `ai_service/`
- **负责前端**：KnowledgeManage.jsx, DocumentManage.jsx
- **文档**：调研报告、软件需求说明书、数据库设计说明书（AI/知识库相关）
- **重点任务**：
  - LLM 适配器设计（抽象基类 + DashScope 实现 + 工厂）
  - RAG 问答流程（向量化→检索→过滤→Prompt→生成）
  - Prompt 模板构建与注入防护
  - 双模式（kb + assistant）问答
  - 知识库 CRUD
  - 文档上传/解析/切块/向量化/入库流水线
  - SQLite + numpy 向量存储
  - 向量检索接口

## 5. 风险点与应对措施

| 风险 | 概率 | 影响 | 应对措施 |
|------|:---:|:---:|----------|
| DashScope API 不可用 | 低 | 高 | 系统提示友好错误，主流业务不依赖AI回归 |
| ChromaDB 兼容性问题 | 高 | 中 | 已迁移到 SQLite + numpy 方案，彻底规避 |
| 服务间联调冲突 | 中 | 中 | 提前定义接口协议，公共依赖由组长统一管理 |
| 进度延迟 | 中 | 高 | 优先保证主链路，扩展功能可降级 |
| 文档与代码不一致 | 中 | 中 | 每日同步检查，文档随代码同步更新 |
| 安全漏洞 | 低 | 高 | 安全加固审查，注入防护、限流、认证全覆盖 |

## 7. 沟通与协作机制

- **每日同步**：每天至少一次短会，同步进度、卡点和计划
- **会议纪要**：记录在项目会议纪要文件中
- **代码管理**：Git 分支协作，公共模块变更需组长 review
- **接口约定**：公共接口提前定义，各服务间通过 HTTP API 通信
