## 3. Gateway 详细设计

### 3.1 模块概述

Gateway 是系统的唯一对外入口，采用 ASGI 中间件栈架构，负责 CORS、请求日志、JWT 鉴权、限流和反向代理转发。

### 3.2 中间件栈

文件：`backend/gateway/middleware.py`

#### 3.2.1 RequestLogMiddleware（请求日志）

**功能**：记录每个请求的方法、路径、状态码、耗时、X-Request-Id。

**流程**：
1. 生成或提取 X-Request-Id
2. 记录请求开始时间
3. 执行后续中间件/路由
4. 记录响应状态码和耗时

#### 3.2.2 JWTAuthMiddleware（JWT 鉴权）

**功能**：验证请求身份，注入用户信息到请求头。

**流程**：
```
提取 Authorization: Bearer <token>
    ↓
检查 Token 是否在黑名单（Redis）
    ↓
解码 JWT，验证签名和有效期
    ↓
提取 user_id, role 信息
    ↓
注入请求头 X-User-Id, X-User-Role
    ↓
传给下游服务
```

**公开路径白名单**：
- `POST /api/user/register`
- `POST /api/user/login`
- `POST /api/user/forgot-password`

**管理员路径**：`/api/knowledge/*`

**JWT 配置**：
- 算法：HS256
- 有效期：24 小时（默认）/ 168 小时（记住我）
- jti：UUID4 hex，支持单 Token 撤销

#### 3.2.3 RateLimitMiddleware（限流）

**功能**：基于 Redis 的滑动窗口限流。

**限流策略**：

| 接口 | 限流 Key | 限制 | 窗口 |
|------|----------|:---:|:---:|
| POST 登录 | IP | 20 次 | 60s |
| POST 注册 | IP | 20 次 | 60s |
| POST AI 问答 | user_id | 10 次 | 60s |
| POST 文档上传 | user_id 或 IP | 10 次 | 60s |

**实现**：Redis `INCR` + `EXPIRE`，超限返回 HTTP 429。

### 3.3 反向代理转发

文件：`backend/gateway/proxy.py`

**路由表**：

| URL 前缀 | 目标 |
|----------|------|
| /api/user | http://localhost:8001 |
| /api/chat | http://localhost:8002 |
| /api/knowledge | http://localhost:8003 |
| /api/ai | http://localhost:8004 |

**特殊处理**：

1. **SSE 流式转发**：POST 消息发送接口，使用 `httpx.stream()` 流式读取下游响应并逐帧 yield
2. **文件上传转发**：POST 文档上传接口，设置 120s 超时，以 `multipart/form-data` 格式透传
3. **普通请求转发**：标准 HTTP 代理，30s 超时

## 4. User Service 详细设计

### 4.1 模块概述

负责用户注册、登录、个人信息管理、密码管理、用户管理（管理员）等功能，是系统的身份域服务。

### 4.2 核心功能设计

#### 4.2.1 用户注册

文件：`backend/user_service/services/user_service.py`

**流程**：
```
1. 校验用户名（2-50 字符，字母数字下划线）
   ↓
2. 校验手机号（11 位，1 开头）
   ↓
3. 校验密码（8-32 字符，含字母和数字，不能包含用户名）
   ↓
4. 检查用户名/手机号唯一性
   ↓
5. bcrypt 哈希密码（rounds=12）
   ↓
6. INSERT INTO users (username, phone, password_hash, role='user')
   ↓
7. 返回成功，自动跳转登录
```

#### 4.2.2 用户登录

**流程**：
```
1. 账号识别：判断输入是用户名还是手机号
   ↓
2. 按 user_id 检查锁定状态（Redis 计数器）
   5 次失败 → 锁定 30 分钟
   ↓
3. 查询用户（按 username 或 phone）
   用户不存在 → 返回"账号或密码错误"
   ↓
4. bcrypt 验证密码
   密码错误 → Redis 计数器 +1，返回"账号或密码错误"
   ↓
5. 检查用户状态（status=1 才可登录）
   ↓
6. 生成 JWT Token（包含 user_id, role, jti）
   ↓
7. 清除 Redis 登录失败计数
   ↓
8. 更新 last_login_at
   ↓
9. 返回 Token + 用户信息
```

#### 4.2.3 JWT Token 管理

**签发**：
```python
payload = {
    "sub": str(user_id),
    "role": role,
    "jti": uuid4().hex,  # 唯一标识
    "exp": now + timedelta(hours=expire_hours),
    "iat": now,
}
```

**黑名单**（退出登录）：
```python
# Redis key: blacklist:{jti}
# TTL = token 剩余有效期
await redis.set(f"blacklist:{jti}", "1", ex=remaining_seconds)
```

文件：`backend/common/dependencies.py`

```python
# require_admin: 要求 role == "admin"
```

## 5. Chat Service 详细设计

### 5.1 模块概述

负责会话管理、SSE 流式消息发送、消息反馈和人工转接工单，是系统的核心业务域服务。

### 5.2 核心功能设计

#### 5.2.1 SSE 消息发送（核心流程）

文件：`backend/chat_service/services/message_service.py`

```
send_message(session_id, user_id, content)
    ↓
1. 校验会话归属
   查询 sessions WHERE session_id=? AND user_id=? AND status=1
   ↓
2. 校验消息内容
   - 长度：≤ 2000 字符
   - 重复字符检测
   ↓
3. 保存用户消息
   INSERT INTO messages (session_id, role='user', content)
   ↓
4. 如果是首条消息，自动设置会话标题
   UPDATE sessions SET title = content[:20]
   ↓
5. 获取对话上下文（最近 20 条消息）
   SELECT * FROM messages WHERE session_id=? ORDER BY created_at LIMIT 20
   ↓
6. 调用 AI Service
   POST http://localhost:8004/api/ai/chat
   请求体：
   {
       "knowledge_base_id": session.kb_id,
       "query": content,
       "mode": session.mode,      // "kb" | "assistant"
       "context": [...],           // 最近 20 条消息
       "top_k": 5
   }
   响应：text/event-stream
   ↓
7. SSE 帧转发（逐 token 透传给客户端）
   帧类型：
   - data: {"type":"token","content":"文"}   // 逐字输出
   - data: {"type":"sources","sources":[...]}  // 引用来源
   - data: {"type":"done"}                      // 流结束
   - data: {"type":"error","code":5001,...}    // 错误
   ↓
8. 流结束后拼接完整 AI 回复
   ↓
9. 保存 AI 消息
   INSERT INTO messages (session_id, role='assistant', content, sources)
```

**SSE 帧序列化**：

文件：`backend/common/sse.py`

```python
# token 帧 - 逐字输出
def token_frame(token: str) -> bytes:
    return f'data: {{"type":"token","content":"{json_escape(token)}"}}\n\n'

# sources 帧 - 引用来源
def sources_frame(sources: list) -> bytes:
    return f'data: {{"type":"sources","sources":{json.dumps(sources)}}}\n\n'

# done 帧 - 流结束
def done_frame() -> bytes:
    return 'data: {"type":"done"}\n\n'

# error 帧 - 错误信息
def error_frame(code: int, message: str) -> bytes:
    return f'data: {{"type":"error","code":{code},"message":"{json_escape(message)}"}}\n\n'
```

```
   ↓
2. INSERT INTO sessions (user_id, kb_id, mode, title='新会话')
   ↓
3. 返回 session_id
```

**会话详情**（含权限控制）：
```
1. 普通用户：只能查看自己的会话
2. 客服/管理员：如果已认领该会话的转接工单，也可查看
```

**工单状态机**：
```
pending（待处理） → claimed（已认领） → resolved（已解决） → closed（已关闭）
```

```
3. SSE 流式发送消息到会话

```

## 6. Knowledge Service 详细设计

### 6.1 模块概述

负责知识库 CRUD、文档上传与管理、文档自动入库（解析→切块→向量化→存储）、向量检索，是系统的内容/资源域服务。

### 6.2 核心功能设计

#### 6.2.1 文档上传与安全校验

文件：`backend/knowledge_service/services/document_service.py`

```
upload_document(kb_id, UploadFile, upload_dir)
    ↓
1. 文件类型校验
   白名单：pdf, docx, txt, md, csv
   MIME type + 扩展名双重校验
   拒绝 .exe, .js, .sh 等可执行文件
   ↓
2. 文件大小校验
   max_size = 20MB (20,971,520 bytes)
   ↓
3. Magic Bytes 校验
   PDF: 文件头 0-4 bytes = b'%PDF'
   DOCX: 文件头 0-4 bytes = b'PK\x03\x04'
   防止扩展名伪装攻击
   ↓
4. 文件名清洗
   移除 .. / \ 路径遍历字符
   仅保留：中文/字母/数字/空格/.-_() （）
   截断过长文件名（≤ 255 字符）
   ↓
5. 同知识库同名文件检查
   SELECT COUNT(*) FROM documents WHERE kb_id=? AND file_name=? AND status='ready'
   ↓
6. 保存文件
   路径：uploads/kb_{kb_id}/{safe_filename}
   重名处理：filename(1).ext, filename(2).ext
   ↓
7. 创建文档记录
   INSERT INTO documents (kb_id, file_name, file_type, file_size, file_path, status='pending')
   ↓
8. 触发异步入库
   schedule_ingest(document_id)
   ↓
9. 返回文档信息
```

#### 6.2.2 文档异步入库流水线

文件：`backend/knowledge_service/services/ingest_service.py`

```
_do_ingest(document_id)  // 在 asyncio.create_task() 中运行
    ↓
1. 查询文档信息
   ↓
2. 按文件类型选择解析器
   ├── txt → text_parser (UTF-8/GBK 编码兼容)
   ├── pdf → pdf_parser (PyMuPDF fitz)
   ├── docx → docx_parser (python-docx)
   ├── md → md_parser
   └── csv → csv_parser (转 "列名: 值" 格式)
   ↓
3. 文本切块
   chunk_text(text, chunk_size=600, overlap=100)
   切分策略：
   - 尽量在句号(。！？\n)处切分
   - 段落边界（连续两个换行）优先
   - 最小块长度 > chunk_size/2
   ↓
4. 调用 AI Service 向量化
   POST http://localhost:8004/api/ai/embedding
   {kb_id, texts: [chunk1, chunk2, ...]}
   响应：{embeddings: [[0.1, ...], [0.2, ...], ...]}
   ↓
5. 写入 SQLite 向量存储
   add_chunks(kb_id, chunks, embeddings, document_id, file_name)
   存储格式：embedding → json.dumps → TEXT 列
   ↓
6. 更新文档状态
   UPDATE documents SET status='ready', chunk_count=N, processed_at=NOW()
   ↓
7. 更新知识库文档计数
   UPDATE knowledge_bases SET document_count = document_count + 1
```

**超时保护**：
- 整体入库：`asyncio.wait_for(120s)`
- 向量化 HTTP 调用：60s 超时（独立 httpx client）
- 超时或失败 → 文档状态标记为 `failed`

**启动恢复**：服务启动时自动查询 `status IN ('pending', 'failed')` 的文档，重新调度入库。

#### 6.2.3 向量存储（SQLite + NumPy）

文件：`backend/knowledge_service/vector_store.py`

**存储表结构**：
```sql
CREATE TABLE vec_chunks (
    kb_id INTEGER NOT NULL,
    document_id INTEGER NOT NULL,
    chunk_index INTEGER NOT NULL,
    file_name TEXT NOT NULL,
    chunk_text TEXT NOT NULL,
    embedding TEXT NOT NULL,  -- JSON 序列化的 float 列表
    PRIMARY KEY (kb_id, document_id, chunk_index)
);
```

**检索算法**：
```python
def search(kb_id, query_embedding, top_k=5):
    1. SELECT * FROM vec_chunks WHERE kb_id=?
    2. 将 query_embedding 转为 numpy array (float32)
    3. 计算余弦相似度 = dot(query, emb) / (|query| * |emb|)
    4. 排序取 top_k
    5. 返回 [{doc_name, score, snippet, document}]
```

**复杂度分析**：
- 时间复杂度：O(N × d)，其中 N 为知识库向量数，d 为维度（1024）
- 空间复杂度：O(N × d × 4) bytes（float32）
- 适合中小规模知识库（< 10 万条向量）

## 7. AI Service 详细设计

### 7.1 模块概述

负责 RAG 流式问答、文本向量化、LLM 适配管理，是系统的 AI 能力域服务，也是唯一直接调用模型厂商的服务。

### 7.2 核心功能设计

#### 7.2.1 LLM 适配器

文件：`backend/ai_service/adapter/`

**抽象基类**：
```python
class LLMAdapter(ABC):
    @abstractmethod
    async def chat_stream(self, prompt: str = None, messages: list = None, **kwargs) -> AsyncGenerator[str, None]:
        """流式对话，逐 token yield"""
        ...

    @abstractmethod
    async def embed(self, texts: list[str]) -> list[list[float]]:
        """文本向量化"""
        ...
```

**DashScope 适配器实现**：
```python
class DashScopeAdapter(LLMAdapter):
    async def chat_stream(self, prompt=None, messages=None, **kwargs):
        enable_search = kwargs.get("enable_search", False)
        if enable_search:
            # 联网搜索：messages 格式 + enable_search=True
            model = "qwen-plus"  # qwen-turbo 不支持搜索
            # search_options: {search_strategy: "max", enable_source: True}
        else:
            # 普通模式：prompt 格式
            model = self.llm_model  # 默认 qwen-turbo

        # 使用 asyncio.to_thread 避免阻塞事件循环
        iterator = await asyncio.to_thread(
            lambda: iter(dashscope.Generation.call(**call_kwargs))
        )

    async def embed(self, texts):
        # 自动分批，每批 ≤ 10 条（DashScope 限制）
        BATCH_SIZE = 10
        for batch in chunks(texts, BATCH_SIZE):
            response = await asyncio.to_thread(
                lambda: dashscope.TextEmbedding.call(model=..., input=batch)
            )
            # 提取 embeddings
```

**工厂模式**：
```python
def create_adapter(config) -> LLMAdapter:
    provider = config.LLM_PROVIDER  # 默认 "dashscope"
    if provider == "dashscope":
        return DashScopeAdapter(api_key, llm_model, embedding_model)
    # elif provider == "openai":
```

#### 7.2.2 RAG 双模式问答流程

文件：`backend/ai_service/services/rag_service.py`

**模式路由**：
```
rag_chat(request, adapter)
    ↓
sanitize_query(query)  // Prompt 注入检测
    ↓
if mode == "assistant":
    → assistant 模式（联网搜索）
else:
    → kb 模式（知识库 RAG）
```

**kb 模式详细步骤**：
```
1. 输入安全清洗
   sanitize_query(query):
   - 14 种注入模式检测（正则）
   - 控制字符移除
   - 重复字符限制
   - 长度限制（≤ 2000）

2. 检查知识库是否为空
   POST /api/knowledge/count → count
   若 count == 0 → 返回 "知识库中暂无任何文档"

3. Query 向量化
   embeddings = await adapter.embed([query])
   query_embedding = embeddings[0]
   超时保护：asyncio.wait_for(30s)

4. 远程向量检索
   POST /api/knowledge/search
   {kb_id, query_embedding, top_k=5}
   返回：[{doc_name, score, snippet, document}]

5. 相似度过滤
   filtered = [item for item in retrieved if item.score >= 0.5]

6. 构建 Prompt
   build_prompt(filtered, context, query, mode="kb"):
   → SYSTEM_PROMPT_KB.format(
       retrieved_chunks=_format_chunks(filtered),
       context=_format_context(context),
       query=query
     )

7. 流式 LLM 生成
   adapter.chat_stream(prompt)
   逐 token yield token_frame(token)
   整体超时：30s

8. 返回 sources + done
   yield sources_frame(sources)
   yield done_frame()
```

**assistant 模式详细步骤**：
```
1. sanitize_query(query)

2. 构建 messages
   build_messages([], context, query, mode="assistant"):
   → [
       {"role": "system", "content": SYSTEM_PROMPT_ASSISTANT},
       ...context_messages,
       {"role": "user", "content": query}
     ]

3. 调用 LLM（联网搜索）
   adapter.chat_stream(
       messages=messages,
       enable_search=True,
       model="qwen-plus"
   )

4. 返回 sources=[] + done
```

**通用助手模式 Prompt**（SYSTEM_PROMPT_ASSISTANT）：

## 8. 异常处理方案

### 8.1 异常分类

| 类别 | 示例 | 处理方式 |
|------|------|----------|
| **业务异常** | 用户名已存在、密码错误、会话不存在 | 返回具体错误码和友好提示 |
| **认证异常** | Token 过期、Token 无效、权限不足 | 返回 401/403，不泄露内部信息 |
| **限流异常** | 超过频率限制 | 返回 429，提示稍后重试 |
| **AI 服务异常** | LLM 超时、向量化失败 | 返回友好提示，标记为 failed |
| **数据库异常** | 连接失败、SQL 错误 | 记录完整日志，返回通用错误 |
| **文件异常** | 文件太大、格式不支持 | 返回具体错误提示 |
| **注入攻击** | Prompt 注入检测命中 | 拒绝请求，记录日志 |

### 8.2 全局异常处理器

文件：`backend/common/exception_handlers.py`

```python
# 业务异常 → 自定义 code + 友好 message
# 验证异常 → 仅返回错误数量，不暴露字段细节
# 通用异常 → 记录完整 traceback，返回"服务器内部错误"
```

### 8.3 AI 服务降级

```

    ↓
```

## 9. 权限与安全设计

### 9.1 权限模型

| 角色 | 权限 |
|------|------|
| **user** | AI 对话、会话管理、消息反馈、转接人工 |
| **agent** | user 权限 + 工单管理、客服消息发送 |
| **admin** | agent 权限 + 知识库管理、文档管理、用户管理 |

### 9.2 安全措施清单

| 层次 | 措施 |
|------|------|
| **传输层** | 内部服务绑定 127.0.0.1，仅网关对外 |
| **认证层** | JWT + jti + Redis 黑名单 |
| **授权层** | require_admin / require_agent 依赖注入 |
| **限流层** | Redis 滑动窗口，IP/用户双维度 |
| **应用层** | Prompt 注入检测、文件名清洗、Magic bytes 校验 |
| **数据层** | bcrypt 密码哈希、SQL 参数化（防注入）、显式列查询 |
| **错误层** | 不泄露内部异常信息 |

## 10. AI 能力调用流程设计

### 10.1 调用链

```
Browser → Gateway → Chat Service → AI Service → DashScope（LLM）
                                    ↓
                              Knowledge Service（向量检索）
```

### 10.2 模型解耦设计

```
业务服务（Chat/Knowledge）
    │ 调用 AI Service 接口
    │ 不关心底层模型
    ▼
AI Service（adapter 层）
    │ 抽象接口：chat_stream() / embed()
    │ 工厂模式创建具体适配器
    ▼
DashScopeAdapter（当前实现）
    │ DashScope SDK
    ▼
阿里云 DashScope（Qwen-Turbo / Qwen-Plus / text-embedding-v3）
```

**切换模型的影响范围**：
- 仅需修改 `adapter/` 目录（新增适配器类 + 修改工厂）
- 业务服务零改动
- 接口不变（chat_stream / embed 签名保持稳定）
