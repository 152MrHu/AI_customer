### 5.3 关键业务流程测试

#### 5.3.1 知识库问答完整流程 ✓

```
操作步骤：
1. 管理员登录 → 创建知识库"测试知识库"
2. 上传 PDF 文档"员工手册.pdf"
3. 等待入库完成（status 变为 ready，chunk_count > 0）
4. 普通用户登录 → 创建会话（选择"测试知识库"，kb 模式）
5. 发送消息："年假政策是什么？"
6. SSE 流式接收回答（逐字显示）
7. 查看 sources（显示引用文档名和相似度）
8. 对回答进行点赞反馈

测试结果：✓ 全流程通过
- 文档入库正常：解析 → 切块 → 向量化 → 存储
- SSE 流式输出正常：token 帧逐字渲染
- sources 帧返回引用的文档名和相似度
- 反馈保存成功
```

### 6.1 测试缺陷分析

|----------|:---:|:---:|:---:|:---:|

| 轻微 | 0 | 0 | 0 | — |

**说明**：在编写文档和开发过程中，通过分层开发和逐一验证的方式，未积累遗留缺陷。

### 6.2 测试经验总结

1. **微服务测试优势**：各服务独立可测，可以先单独验证每个服务的 API，再联调集成，降低了测试复杂度
2. **SSE 流式测试**：SSE 流式接口的测试需要使用支持流式读取的工具（如 curl -N），或编写 SSE 客户端测试脚本
3. **AI 回答准确性测试**：AI 回答具有一定随机性，测试重点是验证系统是否正确基于知识库内容回答（不编造），而非逐字比较
4. **安全测试优先**：将安全测试（认证、鉴权、限流、注入防护）作为必测项，确保系统安全可靠
5. **浏览器兼容性**：在 Chrome 120+ 上测试通过，Edge 兼容，Firefox 需注意 SSE EventSource 的 polyfill
6. **后续建议**：可引入自动化测试框架（如 pytest + httpx），编写 API 测试用例和端到端测试脚本
